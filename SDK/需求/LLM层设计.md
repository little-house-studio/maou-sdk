# LLM层
- LLM层负责：
    - JSON底层
        - json校验、解析
        - json修复、容错
            - 缺失闭合符推断
    - LLM发送消息组装与发送
        - 配置文件
            - 思考深度
            - 模型名称
            - 输入上下文上限
            - 输出上下文上限
        - 临时参数
            - 思考深度
        - 消息结构体
            - 内容（兼容上下文层输入）
            - 多模态附件
        - 其他功能
            - 中断流式
            - 重试发送
    - LLM返回内容消息解析
        - 流式消息组装为非流式消息
        - 思考内容解析
            - 流式和非流式内容解析
        - 工具内容解析
            - 流式和非流式内容解析
        - content内容解析
            - 结构化输出解析
                - 流式和非流式内容解析
            - 流式和非流式内容解析
        - 错误解析
        - usage字段解析（token数、cost等）
            - 流式和非流式内容解析
        - 兼容上下文层输出（写入上下文）
    - 厂商功能
        - 获取可用模型
        - 获取模型上下文
        - 连接测试
    - 厂商协议
        - 厂商协议转内部标准
        - 内部标准转厂商协议
        - 厂商模板预设
        - api配置文件解析
    - 原始POST日志写入
        - 每次 LLM HTTP 调用自动记录标准化日志
        - 单日志管线：只写标准化格式（`event: "llm.post"`）
        - 存储位置：按 agent 分目录，JSONL 追加写入
        - 记录内容：请求摘要（截断 body）、响应文本、token 用量、工具调用摘要、耗时、错误分类
        - 脱敏：Authorization 等敏感 header 替换
        - 错误自动分类（network / timeout / rate_limit / auth / server_error 等）
        - 文件轮转：超阈值自动轮转，保留最近 N 个备份
        - 并发安全：按 sessionId 隔离请求上下文
    - 防BUG
        - OOM循环输出防止
    - SDK
    - 调试接口
        - 计时
            - 首字母计时
            - 生成完毕计时
    - 配置文件与解析
        - 配置文件解析
        - 环境变量注入
