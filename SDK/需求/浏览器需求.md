# 浏览器需求

- 要求
    - 每一次打开网页都是新标签页。
    - 可以获得所有存在的标签页。
    - 每一次执行都要输入标签页页数/id

- 操作方法
    - 1. 打开某个网站
        - 标签页 取名
        - 是否 新标签页 
        - 是否 当前tab目标切换
        - 是否 获得DOM结构树快照/md正文
    - 2. 交互标签页
        - 关闭、开启、切换标签页
    - 3. 网站元素交互
        - 操作类型：
            - 点击元素，输入文本到输入框（并返回该框完整内容），按下按键，滚动（针对ins，微博等需要滑动才能加载更多内容的页面，懒加载页面），
            - 返回页面，选择下拉框，等待元素或文字出现（包括超时时间），执行JS代码。
        - 获取内容类型
            - 获得获得url，某个元素的文字，获得输入框的值，获得元素的所有属性，获得当前网址完整 HTML，获取页面的 API 请求列表，某个请求的完整响应体，页面截图 PNG，执行 JS 返回结果

- 浏览器操作
    1. 打开没有内置Adapter的网站：
        - opencli browser guokr open "https://www.guokr.com/"
        - 为什么browser 「guokr」？
        - 如果是内置Adapter，要什么参数？
        - 如何打开新网页标签？
    2. 打开就得state，看交互元素。
        - opencli browser guokr state
        - 返回完整快照，页面结构：
            - title，viewport，[编号]可点击部分
        - 好像有大小限制？
        - 能不能优化输入内容？
    3. 点击编号部分：
        - opencli browser guokr click 1
        - 返回目标元素是否点击：
            - 是否成功点击，点击的元素，是否有歧义，是否匹配没有漂移。
        - 如何决定是否打开新网页？还是当前框？
        - [ ]是否可以忽略，需要立刻得知点开的内容的消息，失败、新网页内容、成功反馈。
        - [x]如果点击的是别的东西，会返回什么？返回一样的内容。
