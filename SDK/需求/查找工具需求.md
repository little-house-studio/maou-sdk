# 查找工具需求
- sqry二进制打包版本应该分不同的系统
    - windows
    - macos：arm64，x64
    - linux：给出自编译版本

## 需求
- sqry：查找工具code-search
    - 提示词：代码结构搜索工具，类似 grep，但搜的是代码结构（函数、类、调用关系），不是文本。返回符号名+文件路径+行号，不返回原始代码行。能回答 grep 做不到的问题：谁调用了 X、X 依赖了谁、有哪些死代码、有没有循环依赖。
    - 功能参数："search", "callers", "callees", "imports", "exports", "path", "cycles", "unused", "impact"
        - search=按名搜符号，callers=谁调用了它，callees=它调用了谁，imports=谁导入了它，exports=它导出了什么，path=两符号间调用链，cycles=循环依赖，unused=死代码，impact=修改影响范围。
    - symbol：要搜索的符号名
        - 符号名。search/callers/callees/imports/exports/impact 必填。
    - target：目标符号名。path 必填。
    - filter：过滤条件。语言或类型过滤，如 lang:ts、kind:function。
    - limit：最大返回结果数量。默认 30。
- grep：文本内容搜索工具
    - 提示词：用正则搜索文件内容。底层使用 ripgrep，自动处理 .gitignore、二进制文件、编码。output_mode=files_with_matches 只返回文件名（判断有没有）；content 返回匹配行+上下文；count 返回计数。
    - 功能参数："files_with_matches"(默认), "content", "count"
        - files_with_matches=只返回文件名，content=返回匹配行+上下文，count=返回每文件匹配数。
    - pattern：正则表达式。必填。
    - path：搜索目录或文件路径。默认整个项目。
    - glob：限定文件类型，如 '*.py'、'*.{ts,tsx}'。
    - type：rg 内置文件类型过滤，如 'js'、'py'、'rust'。比 glob 更快。
    - ignore_case：忽略大小写。默认 false。
    - multiline：跨行匹配。默认 false。
    - context：匹配行前后各显示 N 行上下文。仅 content 模式有效。
    - context_after：匹配行后显示 N 行。与 context 互斥。
    - context_before：匹配行前显示 N 行。与 context 互斥。
    - head_limit：最大返回行数。默认 250。
- glob：文件名搜索工具
    - 提示词：按文件名模式查找文件。返回匹配的文件路径。自动跳过 .gitignore 中的目录（node_modules、dist 等）。支持递归如 '**/*.py'。
    - pattern：Glob 模式。必填。如 '**/*.py'、'src/*.ts'、'*.md'。
    - path：搜索目录。默认整个项目。
    - head_limit：最大返回文件数。默认 100。
