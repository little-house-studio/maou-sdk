# AI场景
- 终端归属于当前agent,Agent不能看另一个Agent的终端，终端随着agent服务的关闭而安全中断停止，但持久化的内容依旧在，下次启动后终端名单依旧。
- 终端工具3种模式：
    1. 运行任务于终端：
        - 参数：id名，不填id名为临时非持久终端，执行完成就返回结果并销毁终端线程，id名字为ai自己定义的名称
            - 如果是起了一个已存在的ID，那么意思就是对这一个ID的终端进行操作，意思是运行新任务用这一个终端。如果该终端已经有任务在运行，将会提醒ai该终端已经有任务在运行，如有必要可以使用指令停止该任务。
        - 参数：终端指令：要运行的终端指令
        - 参数：任务简介：任务简介，用于ai查看任务状态
        - 参数：前台监听阻塞任务/后台运行任务
            - 前台阻塞任务将会等到内容结束后才会返回工具结果给ai，期间ai什么都做不了只能等待（超时会自动返回所有已有运行结果并且提醒任务xx因为超时转入后台，ai可以使用工具随时查看内容，或者结束该终端id任务）
            - 后台运行任务不会阻塞ai，调用后阻塞1秒内无返回消息将会返回工具信息（任务xx已在后台运行，完成或失败或超时，会自动提醒）
            - 后台任务完成后，会以user信息，并且用XML标签<terminal-message>包裹起来发给ai，这个也相当于属于是一种用户消息，不过是终端后台提醒消息，具体行为让ai自己去决定。
        - 参数：超时时间
            - 后台任务超时会提示ai任务超时，设置为0表示不超时提醒持续在后台常驻
            - 前台阻塞任务超时会自动返回所有已有运行结果，并且自动转入后台，并且提醒ai任务xx因为超时转入后台，ai可以使用工具随时查看内容，或者结束该终端id任务。
        - 参数：返回最后结果的限制字数，0就是不用返回结果数据，只是显示任务完成或失败或超时等明文提醒。
            - 作用是让后台消息可以运行完成，或者失败或者超时，可以直接汇报的时候获取到一定的信息，不需要AI再调工具去查。
            - 另一个作用是减少上下文消耗，AI会对结果有用的内容字数有大概的判断，就可以不需要直接返回完整的结果。
        - 后台任务完成注入时机，如果在1秒内完成，将会立刻返回给AI，继续loop。
            - 如果超过1秒，就自动在后台，并且发送一条消息说这个任务已运行在后台，完成或者失败或者超时就会提醒。
            - 后台任务的注入时机就是直接以user信息发送给AI，让AI自己判断是否进行工具调用，其实就是新的一轮Agent Loop。相当于用户发了条信息给他，或者hook了条信息。所有User消息发送的工具调用结果都是这样，不单只是终端工具这样。
    2.管理类：前后台后台终端管理
        - before_user常驻显示终端状态，不需要调用工具查看终端列表，时刻显示在before_user位置
            - 显示：id，运行的任务简介，运行状态，开始时间，上次消息更新时间，上次查看的时间
            - 时间为月日时分秒
            - 显示完整列表。
        - 删除终端号并且清除该终端内容
            - 如果终端在执行任务，将会提醒ai该终端正在执行任务，如需删除请先停止或等待执行完毕再执行删除指令。
        - 结束终端任务
        - 查看终端内最后xx字的内容（带限制字数，默认5000字）

    4. 注意所有调用工具的后面都应该跟上role:tool的工具返回消息，否则ai会报错。
        - 所以前台阻塞任务超时/成功/失败，这种第一次回应工具调用的都应该以tool消息返回，跟在调用工具的消息后面。
        - 前台终端阻塞中时候除了该阻塞指令工具返回的tool内容，任何消息都不能插入，都应该队列等待。
        - 如果用户操作或者服务掉线等等操作导致工具调用指令后面没有接工具返回消息，就自动插入原因，比如服务器掉线，或者用户强行终止等等。（这个是属于所有工具的通用项，不是终端工具的专属项）
        - 后台任务运行会阻塞一秒等待超快完成的后台任务，或者等待迅速报错，这一段属于返回工具tool返回信息
        - 后台任务如果在后台完成任务或者超时，发送回来的消息内容应该属于role:user的工具返回消息信息，并且使用xml标签<terminal-message>包裹起来通知内容。
        - 所有User信息发送的工具返回结果，都要附带该调用工具的call_ID，以防AI没有理解究竟是哪一个工具指令的结果。相当于tool返回的时候对应了这个call ID，现在这个user也使用这一个call id返回工具通知消息。这个是所有使用User发送工具调用结果的统一标准，并不是终端工具独有的设计。
