# 鼠标方案 Demo 完整需求规格

## 目标
做一个 Ink demo，复现 Claude Code 的鼠标体感：像网页一样，鼠标移动、hover、点击、拖拽选区都正常工作，选区为蓝色背景（程序自画，非终端原生）。支持运行时切换不同鼠标模式，对比选最佳方案。

## 核心功能（所有模式都要的）
1. **渲染区**：6-8 行混合内容（ASCII、中文、emoji、长行 soft-wrap、可点击按钮、缩进文字）
2. **hover 高亮**：鼠标移到按钮上，按钮反色/高亮；移开恢复
3. **点击响应**：点按钮，状态行显示"点击了 X"
4. **拖拽选区**：按住左键拖动，画出蓝色背景选区（实时跟随鼠标）
5. **选区复制**：松手时，选区文本通过 OSC52 写剪贴板，状态行显示"已复制 N 字"
6. **多行选区**：选区可跨行，提取文本正确（CJK 宽字符整组、emoji 整组、soft-wrap 视觉行）
7. **全屏**：进备用屏，从 (1,1) 起，识别终端大小，坐标无需校准

## 模式切换（运行时，按数字键切换）
- **按 1**：模式 A — `?1003` 全追踪 + 程序自画蓝底选区 + OSC52（Claude Code Fullscreen 路线）
- **按 2**：模式 B — `?1002` 按钮拖动追踪 + 程序自画蓝底选区 + OSC52（不报 hover motion，只报按住拖动）
- **按 3**：模式 C — `?1000` 点击释放 + 拖拽时临时关协议让终端原生选区（松手恢复，若收不到 release 则手动恢复）
- **按 4**：模式 D — 不开鼠标协议（终端原生选区，无 hover/点击，对照基准）
- **按 m**：显示当前模式 + 终端大小 + grid 大小
- **按 q / Ctrl+C**：退出

## 选区视觉
- 蓝色背景 + 原色字（`ESC[34;44m` 蓝字蓝底太糊，用 `ESC[97;44m` 亮白字蓝底，或纯 `ESC[44m` 蓝底保留原字色）
- 选区内字符实时重绘为蓝底
- 选区外字符正常
- 松手选区消失（已复制）

## screenBuffer
- 每个可选文本登记到 `Map<"row,col", char>`
- 按 code point 分割（`[...str]`）+ `string-width` 算宽
- soft-wrap：按终端宽度算视觉行，每视觉行登记到对应 row
- 提取选区：按 anchor→focus 行列范围，宽字符整组包含

## 可点击元素
- 按钮 `[选项 A] [选项 B] [选项 C]` 在固定行
- 点击命中：按坐标判断是否在按钮矩形内
- hover 命中：motion 时判断是否在按钮矩形内

## 状态行（底部）
显示：当前模式 | 鼠标坐标 | 事件类型 | hover | 选区范围 | 最后反馈（已复制/点击了X）

## 不做
- 双击选词（后续）
- 跨视口选（scrollback）
- noSelect 标记（demo 全区可选）

## 实现要点
- 纯 node（不用 Ink，避免框架干扰），自己渲染 + 处理 stdin
- 全屏备用屏 + `ESC[H` 定位 (1,1) + 识别 `process.stdout.columns/rows`
- SGR 鼠标解析（`?1006`）
- 选区重绘：拖拽 motion 时重画选区覆盖层（蓝底），不清屏（避免闪烁）
- OSC52：`\x1b]52;c;<base64>\x07`
- 模式切换：发对应 `?100Xh/l` 序列，清屏重绘
