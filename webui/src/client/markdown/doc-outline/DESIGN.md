# 需求文档工作台（PRD Outline）

面向 **写项目需求 / 对齐 PRD** 的人群，不是通用 mindmap。

## 产品目标

1. 快速看清：有多少章节、验收完成度、P0、阻塞  
2. 快速钻取：标题树 → 进章节 → 看验收任务与说明  
3. 写作友好：识别 `REQ-` / `[P0]` / 状态 / `#tag` / `- [ ]`  
4. 与编辑器联动：一键跳到源码行修改  

## 解析能力

| 输入 | 识别 |
|------|------|
| `#`～`######` | 标题树 |
| `REQ-001` `US-12` | 需求 ID |
| `[P0]` `P1:` | 优先级 |
| `[WIP]` `已完成` `阻塞`… | 状态 |
| `#auth` | 标签 |
| 标题含 验收/功能/风险… | 章节角色 |
| `- [ ]` / `- [x]` | 验收任务（汇总进度） |
| 缩进列表 / 代码块 / 引用 / 表格行 | 正文结构树 |

## UI 结构

```
[ 仪表盘：章节数 · 任务 · 完成度 · P0 · 阻塞 ]
[ 搜索 / 优先级 / 状态 / 类型 / 未完成 / 写作技巧 ]
┌─────────────────────────────┬────────────┐
│ breadcrumb · 编辑定位        │ 结构导航   │
│ 当前章节 + 徽章 + 进度条     │ 竖向树     │
│ 需求总览卡片 / 子章节        │            │
│ 正文与验收结构树             │            │
└─────────────────────────────┴────────────┘
```

## 快捷键

| 键 | 行为 |
|----|------|
| `/` | 聚焦搜索 |
| `Esc` | 返回上级章节 |
