# diff
## 变更感知
- 名单更新：
    - 进入名单：如果使用了阅读/编辑/覆写工具操作了某些文件，那么将会把这些文件放进入该会话的diff监听名单，并开始监听轮次倒数。
    - 退出名单，以下方式其中一种生效即退出监听文件名单：
        - 方式1：如果文件连续200轮没有任何阅读/编辑/覆写该文件将会从名单中移除
        - 方式2：如果已经具有5次更改消息发出去，却一次都没有任何阅读/编辑/覆写该文件，第五次更改消息发送后将会从名单中移除。
- 每次用户与agent发送新的对话时，将会检查名单中的文件的diff，并且传递告诉agent有哪些文件有更改了xx行。
    - 要求较高高性能、低内存消耗
    - 内容仅仅告诉agent距离上一次用户发送消息具有以下文件有改动，并且表明距离上次阅读该文件经历了多久时间，改了的文件列表（为了省token，使用相对距今同路劲），不传递.gitignore中的文件。
    - 传递内容时候不要有那种要让agent一定要去看或者让他去看这些文件的提示词，让他自己决定就行。
        - 最好diff消息也加上提示词告诉agent这个可以不需要查看的情况下可以忽略该消息。
    - diff消息放在before_user中，并且以xml包裹。
- 如果发生这种事情：
    1. 用户更改 -> ai阅读/更改文件A -> 用户更改文件A ->用户和ai发送新的一轮信息<message>。
        - 这种情况，<message>里面会有这个文件diff信息。
    2. 用户更改 -> 用户更改文件A -> ai更改/阅读文件A（更改前会提示并被要求阅读文件） ->用户和ai发送新的一轮信息<message>。
        - 这种情况，<message>里面不会有这个文件diff信息，因为最后更改的时候他已经看过了
