# Maou CLI 设计系统 · 磁带未来主义功能框架

## 1. 设计哲学

```
        形式追随功能   ×   磁带录音机时代的触觉记忆
        ─────────────────────────────────────────────
        每一个像素都有理由，每一个操作都有回响。
```

### 核心原则

- **功能主义（Functionalism）**—— 空间为操作服务，装饰只存在于功能间隙。没有纯装饰元素。
- **磁带未来主义（Cassette Futurism）**—— 数据不是无形的云，而是可触碰、可倒带、可暂停的实体。UI 借用 1970-80 年代磁带录音机、CRT 终端、VFD 荧光显示屏的视觉语言。
- **现代人体工学**—— 虽然视觉复古，交互必须符合现代输入设备习惯。无学习成本。

### 用户画像

- **主力用户**：开发者 / Agent 使用者，每天 2-6 小时在 CLI 前
- **辅助用户**：偶尔通过 TUI 管理 Agent 会话的非技术用户

## 2. 磁带未来主义视觉语言

### 2.1 设计参考物

| 参考物 | 提取元素 |
|--------|----------|
| **磁带录音机** | 播放/暂停/快进/停止 图标系统、磁带计数器（七段数码）、VU 表 |
| **CRT 终端** | 绿色/琥珀色等宽字体、扫描线纹理、字符级光标 |
| **VFD 荧光显示屏** | 高对比度发光效果、圆角分段显示、暗色基底 + 亮色文字 |
| **80 年代音响面板** | 旋钮式调节、电平指示灯、铝合金拉丝纹理、实体按键分隔 |
| **Teletext / Ceefax** | 低分辨率块状图形、粗体大字标题、颜色块分区 |

### 2.2 拒绝的元素

- 毛玻璃模糊效果（Frosted glass）
- 圆角 > 6px（磁带时代没有大圆角）
- 阴影（Shadow）—— 用线条替代
- 渐变（Gradient）—— 用色块替代
- Emoji —— 字符集用 ASCII / Unicode 框线字符
- 脉冲动画 > 2 帧 —— 模拟荧光灯启辉器的闪动感

## 4. 空间系统与布局

### 4.1 布局哲学

功能主义的空间利用遵循：

```
              ┌─────────────────────────────────────┐
        HEADER│      操作上下文 + Agent 选择        │ LINE ──
              ├─────────────────────────────────────┤
       BODY   │                                     │
       (70%)  │          主内容区                   │ LINE ──
              │      （消息流 / 对话）               │
              │                                     │
              ├─────────────────────────────────────┤
       STATUS │   Session ID    │ Model │ Tokens    │ LINE ──
              ├─────────────────────────────────────┤
       INPUT  │ > 输入区域                          │
              └─────────────────────────────────────┘
```

- **线条分割，而非留白分割** —— 每个功能区块之间用单线（`─`）或双线（`═`）分隔
- **72-75 列传统终端宽度作为基准** —— 所有内容在 72-80 列内可读
- **垂直空间极度珍惜** —— Header 不超过 2 行，Status 不超过 1 行
- **水平空间** —— Go TUI 随窗口自适应；纯文本 CLI 固定 80 列

### 4.2 区块分隔符号约定

```
══════  ─── 双线：顶层区块分隔（Header ↔ Body）
──────  ─── 单线：子区块分隔（状态栏 ↔ 输入区）
┄┄┄┄┄┄  ┄┄┄ 虚线：折叠提示 / 省略内容
······  ···  点线：加载中 / 等待状态
```

### 4.3 磁带计数器导航

每个会话有一个 "磁带计数器" 形式的定位指示器：

```
Session ───  C-60  ▐█▌ 023:47
              ↑       ↑
           磁带长度   当前位置（第 23 轮，约 47% token 消耗）
```

## 5. 字体与排版

### 5.1 字体层级

| 用途 | 字体 | 说明 |
|------|------|------|
| **等宽正文（主）** | `Berkeley Mono` / `IBM Plex Mono` / `JetBrains Mono` | 代码 + 对话统一等宽 |
| **可变宽度正文** | `Courier Prime`（衬线体，模拟打字机） | 帮助文本、长说明 |
| **七段数码** | `DSEG7 Classic` | 计数器、token 用量显示 |
| **系统消息** | 等宽 + 颜色反转背景 | 区分于对话 |

### 5.2 排版规则

- 消息时间戳：`[HH:MM:SS]` 格式，右对齐，颜色 muted
- Agent 名称：固定宽度前缀，左对齐，如 `[ Vampire ]`（10 字符宽）
- User 输入：无前缀，以 `▸` 开头
- 系统消息：`─── 系统消息 ───` 居中，用等号线包围
- 分隔符统一用 Unicode 框线字符：`┌─┐│└┘├┤┬┴┼`

```
│ ▶ 输出文件: dist/index.js                     │  ← 附件 / 工具结果
└──────────────────────────────────────────────┘
```

### 6.2 输入区（磁带录音键）

```
┌──────────────────────────────────────────────────────┐
│ ▸                                                        │
└──────────────────────────────────────────────────────┘
  REC ○    Pause ⏸    Stop ⏹    Eject ⏏
   ↑
  输入模式指示器（空状态 = 待机 / 闪烁下划线 = 等待输入 / REC = Agent 工作中）
```

### 6.3 工具调用卡片（磁带计数器风格）

```
┌──────────────────────────────────┐
│ TOOL: read                  ████ │  ← 七段数码进度
│ ──────────────────────────────── │
│ file: src/config.ts              │
│ offset: 0                        │
│ ................................................ │  ← 等待中
│ ✓ 读取完成 · 42 lines            │  ← 完成状态
└──────────────────────────────────┘
```

### 6.4 状态栏（VFD 荧光显示屏）

```
═══════════════════════════════════════════════════════
  SESSION d7f3a2e1 │ MODEL deepseek-v3 │ TOKENS 12.4k / 128k │ MODE AUTO
═══════════════════════════════════════════════════════
```

- 每个字段由 `│` 分隔
- Mode 字段：`NORMAL`（绿色） / `AUTO`（琥珀色） / `YOLO`（红色）

### 6.5 命令面板（录音机功能键行）

```
══════════════════════════════════════════
  F1:HELP  F2:NEW  F3:PLAN  F4:TOKEN  F5:VAR
  ← 倒带     → 快进     ↑ 上一会话     ↓ 下一会话
══════════════════════════════════════════
```

## 7. 动效与状态转换

### 7.1 动效原则

| 场景 | 动效 | 时长 |
|------|------|------|
| Agent 开始响应 | 状态栏 Mode 字符闪烁（模拟荧光灯启辉） | 100ms × 2 |
| 消息流式输出 | 逐字符打印（打字机效果） | 0ms（即时）/ 可开关 |
| 工具执行中 | 七段计数器数字滚动 | 持续 |
| 区块切换 | 整行替换，无滑入滑出 | 0ms |
| 错误 | Mode 区域短暂红色闪烁 | 200ms × 2 |
| 加载 | `...` 虚线滚动（`...` → `···` → `····`） | 500ms 循环 |

### 7.2 无动效的场景

- 历史消息加载（直接显示）
- 命令面板展开（瞬间）
- 滚动（依赖终端模拟器）

## 8. 交互模式与用户体验目标

### 8.1 核心交互

| 操作 | 按键 | 视觉反馈 |
|------|------|----------|
| 发送消息 | Enter | 消息立即可见，光标移至输入区 |
| 换行 | Shift+Enter / Alt+Enter | — |
| 取消输入 | Esc | 清空输入区 |
| 中断 Agent | Ctrl+C | 输出 `⏹ STOP` + 闪烁确认 |
| 打开命令面板 | Ctrl+P / / | 功能键行高亮 |
| 滚动历史 | PageUp / PageDown | 计数器变化 |
| 切换会话 | Ctrl+← / Ctrl+→ | Session ID 变化 + VFD 刷新 |
| 复制最后消息 | Ctrl+Y | 无视觉变化（粘贴到输入区） |

### 8.2 用户体验目标

1. **零等待感知** —— 所有操作在 16ms 内给出视觉反馈（不一定是结果）
2. **永不丢失上下文** —— 状态栏始终可见，Session ID 随身
3. **操作可预测** —— 相同按键在不同页面效果一致；/命令 与 功能键 一一对应
4. **倒带式历史追溯** —— 任意会话的消息历史像倒带一样可以快速前后翻阅
5. **触觉记忆模拟** —— 虽然屏幕不可触摸，但动画暗示物理操作：按键 → 弹起、开关 → 到位、计数器 → 进位
6. **渐进式功能暴露** —— 新手只看到输入区和消息，老手知道 Ctrl+P 打开命令面板
7. **区分度优先** —— 不同信息类型（用户消息 / Agent 回复 / 工具调用 / 系统消息 / 错误）有明显的视觉差异，不依赖颜色（色盲友好：靠形状 + 位置区分）

### 8.3 无障碍

- 色盲友好：所有状态同时用符号 + 颜色表示（✓ ✗ ⏳）
- 可配置：字体大小、颜色主题、速度、边框风格均可调
- 屏幕阅读器：通过纯文本 CLI 可直接阅读

## 9. 实现分层

```
┌─────────────────────────────────────────┐
│            Go TUI (Bubble Tea)          │  ← 桌面级 TUI，全视觉语言
│  布局 / 组件 / 动画 / 主题              │
├─────────────────────────────────────────┤
│            TS CLI (纯文本降级)           │  ← 仅用颜色 + 框线字符
│  无布局引擎，纯 stdout                   │
├─────────────────────────────────────────┤
│            共享协议                      │
│  /api/run (SSE) / /api/sessions         │
└─────────────────────────────────────────┘
```

Go TUI 实现全部设计系统组件。TS CLI 作为轻量启动器和构建入口，不提供交互界面。
